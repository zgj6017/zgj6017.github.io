在搭建个人博客的过程中，虽然整体流程比较顺畅，但也遇到了一些意想不到的问题。记录下来，既是为了备忘，也希望能帮到同样遇到这些问题的同学。

## 一、部署时的 SSH 权限问题

### 问题现象
第一次尝试使用 `hexo deploy` 部署到 GitHub 时，终端报错：
```bash
git@github.com: Permission denied (publickey).
fatal: Could not read from remote repository.
```

### 原因分析
1.  **SSH 配置文件名错误**：生成的 SSH 配置文件被命名为了 `config.txt`（带后缀），导致 Git 无法正确读取配置。
2.  **SSH 客户端冲突**：Windows 系统自带的 SSH 客户端与 Git 自带的 SSH 客户端在处理中文路径时存在差异。我的用户名是中文，Git 自带的 SSH 客户端无法正确读取密钥文件。

### 解决方案
1.  将 `config.txt` 重命名为 `config`（无后缀）。
2.  强制 Git 使用 Windows 系统自带的 SSH 客户端：
    ```bash
    git config --global core.sshCommand "C:/Windows/System32/OpenSSH/ssh.exe"
    ```

## 二、部署时的 Host Key 验证失败

### 问题现象
解决了权限问题后，再次部署时卡在确认提示，或者报错：
```bash
Host key verification failed.
```

### 原因分析
这是因为第一次连接 GitHub 服务器时，系统不知道这台服务器是否可信，需要手动确认指纹。自动化脚本无法处理这种交互式询问。

### 解决方案
手动将 GitHub 的服务器指纹添加到 `known_hosts` 文件中：
```bash
ssh-keyscan -H github.com >> ~/.ssh/known_hosts
```

## 三、更换主题时的配置报错

### 问题现象
尝试更换为 `hexo-theme-livemylife` 主题时，生成页面报错：
```bash
TypeError: Cannot read properties of undefined (reading 'forEach')
```
或者提示找不到 `min2read` 函数。

### 原因分析
1.  **配置缺失**：该主题需要将部分配置直接写在站点根目录的 `_config.yml` 中，而不仅仅是主题配置文件。
2.  **插件依赖**：主题使用了字数统计功能，但没有安装 `hexo-wordcount` 插件。

### 解决方案
1.  安装缺失插件：`npm install hexo-wordcount`
2.  将主题所需的配置项合并到站点配置文件中。
3.  （最终方案）考虑到维护成本，决定还是切回更成熟稳定的 `next` 主题。

## 四、自动化部署脚本乱码

### 问题现象
为了方便部署写了一个 `一键部署.bat` 脚本，但运行时中文显示乱码，且命令执行失败。

### 原因分析
Windows 批处理文件（.bat）默认使用 GBK 编码，而编辑器保存为 UTF-8，导致中文字符解析错误，甚至影响了命令行的执行。

### 解决方案
将脚本中的提示信息改为英文，避免编码兼容性问题：
```bat
@echo off
echo Starting Auto Deploy...
call npx hexo clean
call npx hexo generate
call npx hexo deploy
echo SUCCESS!
pause
```

## 五、GitHub 部署任务被取消

### 问题现象
收到 GitHub 发来的邮件，提示 "pages build and deployment: all jobs were cancelled"。

### 原因分析
这是因为在短时间内连续推送了多次代码。GitHub Actions 会自动取消旧的构建任务，只保留最新的一个，以节省资源。

### 结论
这是一个正常现象，不需要处理，只要最后一次部署成功即可。
