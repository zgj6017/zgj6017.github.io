## 一、为什么推荐 MySQL Installer？

MySQL 官方提供了两种安装方式：

- **ZIP 压缩包**：需要手动配置、手动初始化，容易出错
- **MSI 安装包（MySQL Installer）**：图形化界面，自动配置，**特别适合多版本共存**

用 Installer 的最大好处是：它会自动处理好端口、服务名、数据目录的隔离，**你完全不用操心端口冲突的问题**。只要不同时启动两个服务，它们就能和平共处。

------

## 二、准备工作

### 2.1 下载 MySQL Installer

访问 [MySQL 官方下载页面](https://dev.mysql.com/downloads/installer/)，下载 `mysql-installer-community-xxx.msi`。

有两个版本可选：

- **Web 版**：只有几 MB，安装时在线下载组件
- **完整版**：约 300 多 MB，包含所有组件，**推荐下载这个**，避免安装时网络问题

### 2.2 重要提醒：不要点右键！

这是最坑的一个问题：在安装过程中，如果不小心点了鼠标右键，安装窗口会瞬间缩成一条线，完全无法操作。

如果不小心发生了，别慌：

- 在任务栏找到 MySQL Installer 图标
- 右键点击，选择「最大化」
- 或者用快捷键 `Alt + 空格键 + X`

不过最好的方法是**全程用左键点击，别碰右键**。

------

## 三、安装 MySQL 8.0（同时保留 5.7）

### 3.1 启动安装程序

双击下载的安装文件，如果提示需要管理员权限，点击「是」。

### 3.2 选择安装类型

你会看到三个选项：

- **Typical（典型安装）**：安装最常用功能
- **Custom（自定义安装）**：可以自己选择组件和安装路径
- **Complete（完整安装）**：安装所有功能

**推荐选择「Custom」**，因为我们要有选择性地只装需要的组件。

### 3.3 选择要安装的组件

在这个界面，左侧是可选组件，右侧是你要安装的组件。

**只需要添加一项：**

1. 在左侧找到 `MySQL Servers` → `MySQL Server 8.0` → `MySQL Server 8.0.45`
2. 选中后点击中间的绿色箭头 `>`，移到右侧

**⚠️ 注意：不要添加 MySQL Workbench！** 我们后面会用 DataGrip 代替，体验更好。

页面底部有个「Enable the Select Features page」的勾选框，**保持不勾选**，否则后面会多出很多复杂配置。

### 3.4 路径冲突的处理

如果你之前装过 MySQL 5.7，可能会遇到路径冲突的提示：

{% asset_img "屏幕截图 2026-03-05 145317.png" 这是一个图片说明%}

这是因为默认路径（`C:\Program Files\MySQL\MySQL Server 8.0`）可能已被占用。

**处理方法**：

- **方案一（推荐）**：修改安装路径，让 8.0 和 5.7 彻底分开
  - Install Directory：`D:\MySQL\MySQL Server 8.0`
  - Data Directory：`D:\MySQL\Data 8.0`
- **方案二**：确认原文件夹没有重要数据后，直接覆盖（不推荐新手尝试）

### 3.5 开始安装

点击 **Execute**，等待安装完成。这个过程大概 1-5 分钟，取决于电脑性能。

## 四、配置 MySQL 8.0

### 4.1 网络配置（这一步什么都不用改！）

{% asset_img "屏幕截图 2026-03-05 150613.png" 这是一个图片说明%}

**关键提示**：这里**完全不用改端口**，保持默认的 3306 就行。

为什么不用改？

- MySQL Installer 会自动处理好两个版本的隔离
- 只要你**不同时启动两个服务**，就不会有端口冲突
- 平时需要用哪个版本，就启动哪个服务的

| 配置项                | 推荐操作      | 说明                           |
| :-------------------- | :------------ | :----------------------------- |
| Config Type           | 保持默认      | Development Computer（开发机） |
| TCP/IP Port           | **保持 3306** | 不用改！                       |
| X Protocol Port       | 保持 33060    | 不用动                         |
| Open Firewall Ports   | 建议勾选      | 方便后续可能需要的远程连接     |
| Show Advanced Options | 不勾选        | 保持默认                       |

### 4.2 认证方式选择（最容易踩坑的地方！）

{% asset_img "屏幕截图 2026-03-05 153807.png" 这是一个图片说明%}

**一定要选下面那个！**

- ❌ **上面：Use Strong Password Encryption**（MySQL 8.0 默认强加密）
- ✅ **下面：Use Legacy Authentication Method**（兼容 MySQL 5.x）

为什么必须选 Legacy？

1. 你的电脑上已经有 MySQL 5.7，很多现有工具默认使用传统认证
2. 选了 Strong，可能导致 DataGrip 或其他客户端连不上
3. 选了 Legacy，所有工具开箱即用，不用折腾配置

### 4.3 设置 root 密码

{% asset_img "屏幕截图 2026-03-05 153919.png" 这是一个图片说明%}

输入你想要设置的密码，点击 **Check** 检查强度。密码要求：

- 至少 8 位
- 包含大小写字母、数字、特殊字符

**记住这个密码**，以后连接 MySQL 8.0 都要用。

### 4.4 Windows 服务配置

下一步是 Windows 服务配置：

- 服务名：`MySQL80`（会自动和 5.7 的 `MySQL57` 区分开）
- **启动类型**：建议选「Manual」（手动），这样不会开机自启
- 如果之前选过「Automatic」，也没关系，平时不启动就行

点击 **Execute** 应用配置，完成安装。

## 五、为什么不用 MySQL Workbench？

| 对比维度     | MySQL Workbench          | DataGrip                                               |
| :----------- | :----------------------- | :----------------------------------------------------- |
| 支持的数据库 | 仅 MySQL                 | MySQL、PostgreSQL、Oracle、SQL Server 等几乎所有数据库 |
| 智能提示     | 基础                     | ⭐⭐⭐⭐⭐ 极强，上下文感知                                 |
| 多版本管理   | 一般                     | 完美支持，可以同时连接 5.7 和 8.0                      |
| 界面体验     | 传统，部分用户觉得不直观 | 现代化，和 IntelliJ IDEA 风格一致                      |
| 价格         | 免费                     | 付费（可试用 30 天）                                   |

### 5.1 下载 DataGrip

1. 访问 [JetBrains 官网](https://www.jetbrains.com/datagrip/download/)
2. 下载对应系统的版本（有 30 天试用期，学生可免费申请教育版）
3. 安装后打开，界面和 IntelliJ IDEA、PyCharm 风格一致，上手很快

### 5.2 配置 DataGrip 连接两个 MySQL 版本

1. 打开 DataGrip，点击 `+` → `Data Source` → `MySQL`
2. **连接 MySQL 5.7**（需要先启动 MySQL57 服务）：
   - Host：`localhost`
   - Port：`3306`（5.7 默认端口）
   - User：`root`
   - Password：你设的 5.7 密码
   - 点击「Test Connection」测试连接
3. **连接 MySQL 8.0**（需要先启动 MySQL80 服务）：
   - 再点一次 `+` → `Data Source` → `MySQL`
   - 端口同样是 `3306`（因为不同时启动，所以不会冲突）
   - 输入 8.0 的密码
4. DataGrip 会记住这两个连接，你可以根据需要切换
5. 还可以给不同环境设置不同颜色（比如测试库绿色、生产库红色），防止误操作

## 六、如何使用这两个版本？

### 6.1 核心原则：不同时启动

两个版本都用默认端口 3306，所以**不能同时运行**。平时使用遵循这个原则：

1. **查看当前运行的服务**：

   - 打开任务管理器 → 「服务」标签页
   - 或者用管理员身份运行命令提示符：`sc query | findstr MySQL`

2. **启动需要的版本**：

   ```
   # 启动 MySQL 5.7
   net start MySQL57
   
   # 启动 MySQL 8.0
   net start MySQL80
   ```

   

3. **停止当前版本，再启动另一个**：

   ```
   # 停止当前服务
   net stop MySQL57
   
   # 启动另一个服务
   net start MySQL80
   ```

   

### 6.2 设置服务为手动启动（推荐）

为了避免两个服务开机自启导致冲突，建议将两个服务都设为「手动」：

1. 按 `Win + R`，输入 `services.msc`
2. 找到 `MySQL57` 和 `MySQL80`
3. 右键 → 「属性」 → 「启动类型」选择「手动」
4. 点击「确定」

这样电脑开机时不会自动启动 MySQL，需要用哪个再手动启动。

## 七、验证安装

### 7.1 命令行测试连接

**连接 MySQL 5.7**（先确保 MySQL57 服务已启动）：

```
mysql -u root -p
```



**连接 MySQL 8.0**（先停止 5.7，启动 8.0）：

```
mysql -u root -p
```



**注意**：因为端口都是 3306，所以连接命令完全一样，不需要加 `-P` 参数。关键是确保**只有你要连的那个服务在运行**。

### 7.2 DataGrip 中切换连接

在 DataGrip 左侧数据库浏览器中，你可以：

- 同时看到两个连接（即使服务没启动，连接配置也在）
- 双击某个连接，如果对应服务没运行，会提示连接失败
- 启动对应服务后，重新连接即可

## 八、常见问题解决

### 8.1 安装窗口缩得很小看不见

**原因**：不小心点了鼠标右键
**解决方法**：

- 在任务栏找到 MySQL Installer 图标
- 右键点击，选择「最大化」
- 或者按 `Alt + 空格键 + X`

### 8.2 两个服务同时启动了怎么办？

如果不小心把两个 MySQL 服务都启动了，后启动的那个会报错「端口已被占用」。

**解决方法**：

1. 打开任务管理器 → 「服务」标签页
2. 找到其中一个 MySQL 服务，右键点击「停止」
3. 只保留你想用的那个版本

### 8.3 客户端连接 8.0 报错 "Authentication plugin 'caching_sha2_password'"

**原因**：安装时选了强加密认证，但客户端不支持
**解决方法**：

- 如果还没装完：退回上一步，选 Legacy 认证

- 如果已经装完：修改用户认证方式

  ```
  ALTER USER 'root'@'localhost' IDENTIFIED WITH mysql_native_password BY '你的密码';
  ```

  

### 8.4 忘记了 root 密码怎么办？

如果不小心忘了密码，可以用这个步骤重置：

1. 停止 MySQL 服务
2. 用 `--skip-grant-tables` 模式启动
3. 修改密码

具体操作网上有很多教程，建议把密码记在安全的地方。

## 九、总结

用 MySQL Installer 安装多版本，关键点就这几个：

1. **下载 MySQL Installer**，用 Custom 模式
2. **只装 Server，不装 Workbench**（后面用 DataGrip）
3. **端口完全不用改**，保持默认 3306
4. **不同时启动两个服务**，用哪个启哪个
5. **认证方式选 Legacy**，避免连接问题
6. **千万别点右键**，不然窗口会缩得看不见
7. **用 DataGrip** 替代 Workbench，体验更好

这套方案的好处是：

- ✅ 安装简单，不用手动配置端口
- ✅ 使用灵活，想用哪个版本就启动哪个
- ✅ 管理方便，DataGrip 统一管理两个连接
- ✅ 学习成本低，不用记两个端口号
